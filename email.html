Introduction To Barrel Strength Email Development
(Use this template as a starting point for email development!)


There are a few basic things to note about our development style:
  • We try to keep documents as "loose" as possible. If it's possible to use stacked tables instead of tr tags, please do. If possible, stay away from heavily nested code (Nested tables are sometimes necessary, but often can be avoided, resulting in cleaner and less buggy code).
  • We try to organize our CSS within the document Everything is inlined when we upload our document, so focus is on readability during development. Remember that a single email may go through many revisions, and be worked on by multiple developers before it is finally blasted to our recipients.
    • Verbose, descriptive names are great.
    • Comments are great. The more, the better. (We'll strip them out before blasting anything.)
    • We'll give images descriptive names -- within reason. Underscores can be your friend!
    • We use spaces, tabs and new lines in code to keep things readable -- if it doesn't need to be on one line, it probably looks better spread out.
    • We split CSS into sections:
      1. First, there are the reset and utility styles.
      2. Then the IDs which we mostly apply to Tables and use to define background color. We also add IDs to each table -- even without styling -- as a short description of the table (these are stripped at upload but make development easier).
      3. After the IDs we have the fonts, starting with global fonts and moving down the document in order of appearance. We keep font styles separate from layout styles -- even when the same class is used -- for the sake of organization and readability.
      4. After the font styles come the layout styles (padding, width, border, background, etc.), in order of appearance.
      5. Finally, we have the mobile-specific styles. In many cases the email is not conducive to responsive design... Don't worry about these styles, we'll add them to the code or delete them as necessary before uploading.
  • For all links we use 'target=_blank', including Campaign Monitor-specific tags unsubscribe and webversion.
  • We close our image tags and breaks!
That's it. Thanks for working with us!
